JavaScript Variables

In JavaScript, variables are used to store data that can be referenced and manipulated later in a program. Variables act as containers for values and are essential for writing dynamic and interactive code.

Declaring Variables

JavaScript provides three ways to declare variables:

  • var: The old way to declare variables, introduced in ES5 and earlier.
  • let: A modern way to declare variables, introduced in ES6, with block scope.
  • const: Used to declare constants whose values cannot be reassigned, also introduced in ES6.

General Example of Variables

Below is a general example demonstrating variable declaration and usage:

// Declaring variables
var firstName = "Alice";
let age = 25;
const country = "USA";

// Using variables
console.log("Name:", firstName); // Output: Name: Alice
console.log("Age:", age);        // Output: Age: 25
console.log("Country:", country); // Output: Country: USA

// Updating variables
firstName = "Bob";
age = 30;
console.log("Updated Name:", firstName); // Output: Updated Name: Bob
console.log("Updated Age:", age);        // Output: Updated Age: 30

// Trying to update a constant
// country = "Canada"; // Error: Assignment to constant variable

1. var

The var keyword was the original way to declare variables in JavaScript. It has function scope or global scope but no block scope.

// Example using var
var name = 'Alice';
console.log(name); // Output: Alice

if (true) {
    var age = 25;
}
console.log(age); // Output: 25 (var does not have block scope)

Key Points about var:

  • Variables declared with var are hoisted and initialized with undefined.
  • They can be re-declared and updated within the same scope.
  • It is generally recommended to avoid using var in modern JavaScript.

2. let

The let keyword is used to declare variables with block scope. This means the variable exists only within the enclosing block.

// Example using let
let name = 'Bob';
console.log(name); // Output: Bob

if (true) {
    let age = 30;
    console.log(age); // Output: 30
}
// console.log(age); // Error: age is not defined (block scope)

Key Points about let:

  • Variables declared with let are hoisted but not initialized, causing a ReferenceError if accessed before declaration.
  • They cannot be re-declared in the same scope but can be updated.
  • Use let when the variable's value needs to change.

3. const

The const keyword is used to declare constants. These are variables whose values cannot be reassigned. Like let, const has block scope.

// Example using const
const name = 'Charlie';
console.log(name); // Output: Charlie

// name = 'Dave'; // Error: Assignment to constant variable

if (true) {
    const age = 35;
    console.log(age); // Output: 35
}
// console.log(age); // Error: age is not defined (block scope)

Key Points about const:

  • Constants must be initialized at the time of declaration.
  • The value of a constant cannot be updated or re-declared.
  • While the constant itself cannot be reassigned, its contents (if an object or array) can be modified.
// Example of modifying a const object's properties
const person = { name: 'Eve', age: 40 };
person.age = 41; // Allowed
console.log(person); // Output: { name: 'Eve', age: 41 }

Differences Between var, let, and const

Feature var let const
Scope Function or global Block Block
Reassignment Allowed Allowed Not allowed
Hoisting Yes (initialized to undefined) Yes (not initialized, gives ReferenceError if accessed before declaration) Yes (not initialized, gives ReferenceError if accessed before declaration)

By understanding the differences between var, let, and const, you can choose the appropriate keyword based on your requirements and write cleaner, more maintainable code.
